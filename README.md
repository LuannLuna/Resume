## How It Works

1. **Source of truth**: All content lives in `resume-en.md`
2. **LaTeX generation**: When the markdown is updated, `latex/resume-en.tex` (English) and `latex/resume-pt.tex` (Portuguese) are regenerated to match
3. **Local build**: Run `make all` inside `latex/` to compile both PDFs
4. **CI/CD**: Any push to `main` that touches `latex/**` triggers GitHub Actions, which compiles both `.tex` files, commits the resulting PDFs to `pdf/`, and GitHub Pages serves the landing page

## Making Updates

1. Edit `resume-en.md` (the source of truth)
2. Regenerate the `.tex` files — both EN and PT — to reflect the changes
3. Build locally to verify:
   ```bash
   cd latex
   make all        # build both versions
   make en         # English only
   make pt         # Portuguese only
   make clean      # remove build artifacts
   make log        # show build logs for debugging
   ```
4. Commit and push:
   ```bash
   git add resume-en.md latex/resume-en.tex latex/resume-pt.tex
   git commit -m "Update resume content"
   git push
   ```
5. GitHub Actions builds the PDFs automatically and commits them to `pdf/`

## Building Locally

### Prerequisites

A LaTeX distribution with XeLaTeX:

```bash
# Full distribution (recommended)
brew install --cask mactex

# Minimal distribution
brew install --cask basictex
sudo tlmgr install enumitem titlesec hyperref fontspec polyglossia
```
